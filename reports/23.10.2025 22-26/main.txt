Импортируются необходимые модули: asyncio для работы с асинхронным программированием, os для управления операционной системой, threading для создания потоков, dotenv для загрузки переменных окружения, а также BusinessBot и app из соответствующих модулей. Эти импорты необходимы для функционирования приложения, так как они предоставляют необходимые функционалы, такие как работа с окружением, асинхронность, и создание веб-сервера и бота.

Функция run_bot отвечает за настройку окружения и запуск бота в асинхронном режиме. Сначала выполняется функция setup_environment, которая подготавливает необходимые переменные окружения. Затем создается экземпляр класса BusinessBot, после чего вызывается метод run_async, который запускает логику бота. Это важно для обеспечения корректной работы бота, который будет обрабатывать запросы пользователей.

Функция run_web отвечает за запуск веб-сервера на указанном порту. Она считывает переменную окружения PORT, которая определяет, на каком порту будет запущен сайт, после чего эта информация выводится в консоль. Затем вызывается метод run приложения, который инициирует веб-сервер. Эта функция важна, поскольку она обеспечивает доступ пользователей к веб-интерфейсу приложения.

Блок `if __name__ == "__main__":` служит точкой входа для выполнения программы. Внутри этого блока сначала выводится сообщение о запуске приложения. Затем создается и запускается новый поток, который выполняет функцию run_web, позволяя веб-сайту работать параллельно с основным потоком. Наконец, основной поток переходит к запуску функции run_bot, которая отвечает за работу бота. Этот блок необходим для инициации всей логики приложения и его правильного запуска в многопоточном режиме.
