# План разработки утилиты AUTO

Этот документ описывает шаги по доработке и исправлению утилиты `AUTO.py` для автоматического документирования кода с помощью ИИ.

### Новые правила разработки:

1.  **Не ставить много комментариев.** Использовать их только для действительно неочевидных частей кода.
2.  **Не удалять существующий код.** Если нужно временно отключить часть функционала (например, для теста), его следует комментировать, а не удалять.
3.  **Не делать отчеты по `.md` файлам.**

---

### 1. Стабилизация работы с ИИ (Решение проблемы `har_and_cookies`)

- **Проблема:** Ошибка `har_and_cookies dir is not readable` вызвана тем, что `g4f` по умолчанию может выбирать провайдера, требующего наличия браузера.
- **Решение:** Жестко указать в коде стабильного провайдера, который не требует браузерных данных. Например, `g4f.Provider.FreeGpt` или `g4f.Provider.Llama3`. Это устранит текущую ошибку и сделает работу утилиты предсказуемой.
- **Действие:** Внести изменения в вызов `g4f.ChatCompletion.create`, добавив параметр `provider`.

### 2. Гибкая настройка параметров

- **Проблема:** Сейчас все параметры (расширения файлов, исключенные папки, промпт) жестко прописаны в коде.
- **Решение:** Вынести все настраиваемые параметры в начало файла в виде отдельного конфигурационного блока или класса.
- **Действия:**
    1.  Создать в `AUTO.py` класс `Config` или просто словарь.
    2.  Перенести туда:
        -   `file_extensions`: список расширений для поиска (например, `['.py', '.js', '.md']`).
        -   `excluded_dirs`: список папок для исключения.
        -   `prompt`: текст промпта для ИИ.
        -   `max_workers`: количество одновременных потоков.

### 3. Модификация логики поиска и фильтрации файлов

- **Проблема:** Скрипт ищет только `.py` файлы.
- **Решение:** Изменить функцию `main`, чтобы она искала файлы со всеми расширениями, указанными в новой конфигурации.
- **Действия:**
    1.  Настроить `glob` для поиска файлов по списку расширений.
    2.  Убедиться, что логика исключения папок и самого файла `AUTO.py` продолжает работать корректно.

### 4. Улучшение системы отчетов

- **Проблема:** Текущая система работает, но можно сделать ее более информативной.
- **Решение:** Добавить в лог больше информации о процессе работы.
- **Действия:**
    1.  Выводить в консоль список найденных и отфильтрованных файлов перед началом обработки.
    2.  В случае ошибки записывать в лог не только сообщение об ошибке, но и имя файла, который не удалось обработать.
    3.  По завершении выводить сводную статистику: сколько файлов обработано успешно, сколько — с ошибками.

### 5. Документация и чистота кода

- **Проблема:** Для нового пользователя может быть не очевидно, как пользоваться скриптом.
- **Решение:** Добавить комментарии и следовать стандартам PEP 8.
- **Действия:**
    1.  Добавить комментарии к конфигурационному блоку, объясняющие каждый параметр.
    2.  Проверить код на соответствие PEP 8 и отформатировать его для лучшей читаемости.
